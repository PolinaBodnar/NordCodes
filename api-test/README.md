Результаты API тестирования
Содержание архива:
1.  Тест план.docx  - Тест план с приоритетами и ожидаемыми результатами
2. Чек лист.docx  - Список проверок по тест плану
3.  api-test-project.json - Рабочая коллекция Postman с тестами
4.  Баг репорты.docx - Баг репорты с описанием найденных дефектов

Как использовать:
1. Импортируйте api-test-project.json в Postman
2. Создайте Environment в Postman
3. Выполните тесты по порядку
4. Изучите найденные баги в Баг репорты.docx (Сейчас тесты исправлены)
 
Найденные баги:
001: Неправильный регистр action (ожидается "withdraw", но API требует "WITHDRAW")
002: Неправильный формат числа (ожидается 10.00, но API требует 10)
003: Противоречие в спецификации

Рекомендации:
Используйте обходные решения из коллекции для успешного выполнения тестов. (Сейчас тесты исправила, баги нашла и зафиксировала)
